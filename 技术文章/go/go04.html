# 一、 接口

# 1 接口的定义

接口是一种类型，是一种特殊的类型，它规定了变量有哪些方法，但是没有具体实现，由其他类型（自定义类型）实现。

type 接口类型名 interface {
    方法名1( 参数列表1 ) 返回值列表1
    方法名2( 参数列表2 ) 返回值列表2
    …
}

补充：接口与结构体的区别

  • 接口是一种类型，规定了变量有哪些方法，但是没有具体实现，由其他类型（自定义类型）实现。

  • 结构体是一种类型，是由一系列具有相同类型或不同类型的数据构成的数据集合。

  • 结构体用于组织和存储数据，接口用于定义行为规范(约束行为、方法)，实现多态。 结构体是数据的实体描述，接口是行为的抽象描述。

总的来说，结构体用于表示数据的结构，而接口用于表示行为的规范。在实际应用中，结构体和接口经常一起使用，结构体用于表示具体的数据结构，而接口则用于定义操作这些数据的方法规范。

# 2 接口的实现与使用

  1. 一个变量如果实现了接口中规定的所有方法，那么这个变量就实现了这个接口，可以称为这个接口类型的变量。

    接口保存的分为两部分：值的类型和值本身。如果接口内的值为 nil，那么其类型也为 nil。这样就实现了接口变量能够存储不同的值。

  2. 具体示例：

package main

import "fmt"

// TODO: 定义一个接口 车
type car interface {
	run() //只要实现了run方法的类型都是实现了car接口，方法签名
}

type wenjie struct {
	brand string
}

func (w wenjie) run() {
	fmt.Println(w.brand, "速度700迈")
}

type xiaomi struct {
	brand string
}

func (x xiaomi) run() {
	fmt.Println(x.brand, "速度700迈")
}

func drive(c car) {
	c.run()
}

func main() {
	var a car
	var w = wenjie{
        // TODO: 定义一个wenjie结构体
		brand: "问界",
	}
	a = w //接口类型的变量能够存储所有实现了该接口的实例
	fmt.Println(a) // {问界}
	a.run() // 问界 速度700迈
	var x = xiaomi{
		brand: "小米",
	}
	drive(w)
	drive(x)
}

# 3 使用值接受者与使用指针接受者实现接口的区别

  1. 使用值接受者实现接口，结构体类型和结构体指针类型的变量都能存储到接口变量中。

  2. 使用指针接受者实现接口，只有结构体指针类型的变量才能存储到接口变量中。

具体代码：

package main

import "fmt"

type animal interface {
	move()
	eat(string)
}

type cat struct {
	name string
	feet int8
}

// 使用值接受者实现了接口的所有方法

func (c cat) move() {
	fmt.Println("走猫步")
}

func (c cat) eat(food string) {
	fmt.Printf("猫吃%s\n", food)
}

// 使用指针接受者实现了接口的所有方法

type dog struct {
	feet int8
}

func (d *dog) move() {
	fmt.Println("汪汪汪")
}

func (d *dog) eat(food string) {
	fmt.Printf("狗吃%s\n", food)
}

func main() {
	var a1 animal // 定义一个接口类型的变量
	c1 := cat{
		name: "tom",
		feet: 4,
	}
	d1 := &dog{
		feet: 4,
	}
	a1 = c1 // 实现了animal这个接口的是cat的指针类型
	fmt.Println(a1)
	a2 = d1 // 实现了animal这个接口的是dog的指针类型
	fmt.Println(a2)
}

# 4. 空接口

  1. 空接口是指没有定义任何方法的接口。因此任何类型都实现了空接口。

  2. 作用：

    • 空接口类型可以保存任意值的类型。
    • 空接口类型作为函数的参数，可以接受任意类型的值作为参数。
    • 空接口可以作为 map 的 value。

  3. 格式：

interface{}  //空接口类型  可以不用命名
1
  1. 示例：
package main

import "fmt"

// 空接口作为函数参数
func empty(a interface{}) {
	fmt.Printf("type:%T value:%v\n", a, a)
}

func main() {
	var m1 map[string]interface{}
	m1 = make(map[string]interface{}, 16)
	m1["name"] = "小王子"
	m1["age"] = 18
	m1["merried"] = true
	m1["hobby"] = [...]string{"看书", "跳舞", "跑步"}
	fmt.Println(m1)

	empty(false)
	empty(nil)
	empty(m1)
}

# 5. 类型断言

  1. 语法：
x.(T)  // x：表示类型为 interface{} 的变量  T：表示断言 x 可能是的类型。 检查空接口变量 x 中保存的值是否为 T 类型。
1

  1. 说明：

    • x：表示类型为 interface{} 的变量
    • T：表示断言 x 可能是的类型。

  2. 作用：

    • 检查空接口变量 x 中保存的值是否为 T 类型。
    • 如果 x 中保存的值的确为 T 类型，则返回其值给变量 ok，否则返回一个 panic。

  3. 示例：

package main

import "fmt"

func assign(a interface{}) {
	fmt.Printf("%T\n", a)
	str, ok := a.(string)
	if !ok {
		fmt.Println("猜错了")
	} else {
		fmt.Println("传进来的是一个字符串：", str)
	}
}

func main() {
	assign(100)
	assign("hello")
}
