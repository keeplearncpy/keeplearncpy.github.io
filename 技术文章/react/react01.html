# 一、 React简介

# 1. 什么是React

  • React是一个用于构建用户界面的JavaScript库
  • 官网：https://reactjs.org/
  • React主要用于构建UI，很多人认为React是MVC中的V（视图）
  • React是单向数据流

# 2. React特点

  • 声明式设计：React采用声明式设计，可以轻松描述应用
  • 高效：React通过对DOM的模拟，最大限度地减少与DOM的交互
  • 灵活：React可以与已知的库或框架很好地配合
  • JSX：JSX是JavaScript的语法扩展，React使用JSX来描述用户界面
  • 组件化：通过React构建组件，使得代码更加容易得到复用，能够很好地应用在大项目的开发中
  • 具有跨平台能力，多平台适配(React Native, ReactVR 虚拟现实技术)

# 3. 开发React必须依赖的三个库

  • react.js：提供了组件化开发所需的核心API (react核心代码)
  • react-dom.js：提供了与DOM相关的功能 (react渲染在不同平台所需要的核心代码)
  • babel.js：将JSX语法转换为JavaScript语法

# 3.1 为什么要进行拆分

就是因为react-native，react包包含了react web 和react-native所拥有的核心代码，所以需要进行拆分

# 3.2 Babel.js

  • Babel是一个广泛使用的转码器，可以将ES6代码转为ES5代码，从而在现有环境执行
  • Babel可以转换语法，比如将ES6的箭头函数转换为ES5的函数，也可以转换新的API，比如Generator、Set、Maps、Proxy等
  • 在react项目中，我们可以一致编写JSX语法，并且让babel帮组我们转换成React.createElement()语法

# 3.3 如何引入react

  • 方式一： 直接CDN引入
<script crossorigin src="https://unpkg.com/react@18/umd/react.production.min.js"></script>
<script crossorigin src="https://unpkg.com/react-dom@18/umd/react-dom.production.min.js"></script>
<script src="https://unpkg.com/babel-standalone@6/babel.min.js"></script>

# 二、JSX语法

# 1. 什么是JSX

const element = <div>hello world</div>
const root = ReactDOM.render(div, document.getElementById('root'))
root.render(element)
1
2
3
  • JSX是JavaScript的语法扩展，长得像HTML，但并不是HTML
  • 它用于描述我们的UI界面，并且可以和js融合在一起使用
  • JSX在编译的时候会被转换为普通的JavaScript对象
  • 为什么React选择JSX
    • React认为，UI和逻辑应该是紧密耦合在一起的，既然使用了React来构建UI，那么逻辑也应该使用JSX来构建
    • JSX可以很好地描述UI应该呈现出它应有交互的本质形式
    • JSX在编译为JavaScript代码后进行了优化，执行更快
    • JSX是类型安全的，在编译过程中就能发现错误
    • JSX编写模板更加简单快速

# 2. JSX语法规则

  • JSX语法的本质：还是在运行的时候，被转换成了React.createElement形式来执行的
  • 书写规范：
    • jsx结构中只能有一个根元素
    • jsx结构通常会包裹一个（），将整个jsx当作一个整体，实现换行
    • jsx结构中，如果要写注释，只能这样写{/这是注释/}

# 3. JSX嵌入变量作为子元素

  • 在jsx语法中，如果想要嵌入变量，需要使用{}来包裹变量
  • 当变量是Number、String、Array类型时，会直接显示
  • 当变量是Boolean、Null、Undefined时，不会显示，内容为空
    • 如果希望显示null、Boolean或者undefined，需要先转换为字符串
    • 转换为字符串的方法：String(null)、String(true)、String(undefined)，toString（）和空字符串拼接也可以
  • 当变量是对象时，会报错，因为对象不能直接显示在页面上(not vaild as a React child)(可以使用JSON.stringify()转换为字符串)

# 4. JSX中可以插入表达式

  • 在jsx语法中，可以在{}中插入任意的表达式
  • 运算表达式
  • 三元运算符
  • 执行一个函数

# 6.JSX中的事件绑定

  • this绑定的四种规则

    • 1.默认绑定 独立执行 函数内部this指向window 严格模式下指向undefined foo()
    • 2.隐式绑定 函数作为对象的方法调用 函数内部this指向该对象 obj.foo() -> foo函数内部this指向obj
    • 3.显示绑定 函数通过call apply bind调用 函数内部this指向指定的对象 foo.call(obj) -> foo函数内部this指向obj
    • 4.new绑定 构造函数调用 函数内部this指向新创建的对象

  • this绑定方式1：在构造函数中显示绑定

  • this绑定方式2：使用箭头函数 ES6 class fields语法

  • this绑定方式3：直接传入一个箭头函数

  • 事件参数传递

  • 最好的方式就是传入一个箭头函数，主动执行的事件函数，并且传入相关的其他参数

# 7.JSX中的条件渲染

class App extends React.Component {
        constructor() {
            super()
            this.state = {
                message: 'hello world',
                isLogin: true,
                obj:{
                    name:'张三',
                    age:18
                }
            }
        }
        onLogOut() {
            this.setState({
                isLogin: false
            })
        }
        onLogin() {
            this.setState({
                isLogin: true
            })
        }
        render() {
            const { message, isLogin,obj } = this.state
            let showElement = null
            if (isLogin) {
                showElement =
                    <div>
                        <h1>{message}</h1>
                        <button onClick={() => this.onLogOut()}>退出</button>
                    </div>
            } else {
                showElement =
                    <div>
                        <h1>请登录</h1>
                        <button onClick={() => this.onLogin()}>登录</button>
                    </div>
            }
            return (
                <div>
                    {/*方式一：根据条件给变量赋值给不同的内容*/}
                    {showElement}
                    {/*方式二：三元运算符*/}
                    <div>{isLogin ? <h2>退出</h2> : <h2>登录</h2>}</div>
                    {/*来回切换性能优化*/}
                    <h2 style={{display: isLogin ? 'block' : 'none'}}>css性能优化</h2>
                    {/*方式三：逻辑与运算符*/}
                    <div>{obj && <div>{obj.name + '' + obj.age}</div>}</div>
                </div>
            )

        }
    }
    const app = ReactDOM.createRoot(document.getElementById('root'))
    app.render(<App />)

# 三、 JSX本质

# 1. 本质

  • JSX语法的本质：还是在运行的时候，被转换成了React.createElement形式来执行的
  • createElement传递的三个参数：
  • 参数1：type
    • 当前ReactElement的类型；
    • 如果是标签元素，那么就使用字符串表示"div";
    • 如果是组件元素，那么就直接使用组件的名称；
  • 参数2： config
    • 所有的JSX的属性都在config中一对象的属性和值的形式存储；
    • 比如传入className作为元素的class；
  • 参数3： children
    • 存放在标签中的内容，以children的数组的方式进行存储；

# 2. 虚拟DOM

  • 作用：
  1. 使用diff算法进行差异更新，不做整个页面更新。
  2. 在React Nactive中，虚拟DOM可以进行跨平台开发，在web端生成div，button的标签，在ios/android端生成UIView，UIButton等标签。
  3. 虚拟DOM帮组我们从命令式编程转到了声明式编程的模式。
  4. 我们可以通过React.render 让虚拟DOM和真实DOM同步起来，这个过程叫做协程(Reconciliation)
  • 这种编程的方式赋予了React声明式的API
    • 你只需要告诉React希望让UI是什么状态；
    • React来确保DOM和这些状态时匹配的；
    • 你不需要直接进行DOM操作，就可以从手动更改DOM、属性操作、事件处理中解放出来；

# 2. React组件化开发

  • 组件化开发的好处
    • 可以让我们的代码更加模块化，也就是说一个组件的代码只负责一个功能
    • 方便复用组件
    • 方便团队开发，提高开发效率
    • 方便维护，Bug少
  • 组件化可以分为类组件与函数式组件

# 2.1 类组件(如何定义一个类组件)

- 1. 定义一个类（类名大写，组件的名称必须是大写的，小写的被认为是html元素），继承自React.Component
- 2. 在类中，必须定义一个render函数，render函数中，必须返回一个合法的JSX虚拟DOM结构

// 1. 创建组件的方式一：构造函数
// 2. 在组件中，必须返回一个合法的JSX虚拟DOM元素
// 3. 组件名称首字母必须大写
// 4. 组件的调用与原生的HTML标签一样，区别在于组件的调用是以标签的形式进行调用的
// 5. 组件的属性与原生的HTML标签一样，区别在于组件的属性值，都是字符串
// 6. 组件的属性值可以在组件内部通过this.props来获取
 // 类组件

        }
    }
    const app = ReactDOM.createRoot(document.getElementById('root'))
