# 一、 React简介

# 1. 什么是React

  • React是一个用于构建用户界面的JavaScript库
  • 官网：https://reactjs.org/
  • React主要用于构建UI，很多人认为React是MVC中的V（视图）
  • React是单向数据流

# 2. React特点

  • 声明式设计：React采用声明式设计，可以轻松描述应用
  • 高效：React通过对DOM的模拟，最大限度地减少与DOM的交互
  • 灵活：React可以与已知的库或框架很好地配合
  • JSX：JSX是JavaScript的语法扩展，React使用JSX来描述用户界面
  • 组件化：通过React构建组件，使得代码更加容易得到复用，能够很好地应用在大项目的开发中
  • 具有跨平台能力，多平台适配(React Native, ReactVR 虚拟现实技术)

# 3. 开发React必须依赖的三个库

  • react.js：提供了组件化开发所需的核心API (react核心代码)
  • react-dom.js：提供了与DOM相关的功能 (react渲染在不同平台所需要的核心代码)
  • babel.js：将JSX语法转换为JavaScript语法

# 3.1 为什么要进行拆分

就是应为react-native，react包包含了react web 和react-native所拥有的核心代码，所以需要进行拆分

# 3.2 Babel.js

  • Babel是一个广泛使用的转码器，可以将ES6代码转为ES5代码，从而在现有环境执行
  • Babel可以转换语法，比如将ES6的箭头函数转换为ES5的函数，也可以转换新的API，比如Generator、Set、Maps、Proxy等
  • 在react项目中，我们可以致谢编写JSX语法，并且让babel帮组我们转换成React.createElement()语法

# 3.3 如何引入react

  • 方式一： 直接CDN引入
<script crossorigin src="https://unpkg.com/react@18/umd/react.production.min.js"></script>
<script crossorigin src="https://unpkg.com/react-dom@18/umd/react-dom.production.min.js"></script>
<script src="https://unpkg.com/babel-standalone@6/babel.min.js"></script>

# 二、JSX语法

# 1. 什么是JSX

const element = <div>hello world</div>
const root = ReactDOM.render(div, document.getElementById('root'))
root.render(element)
1
2
3
  • JSX是JavaScript的语法扩展，长得像HTML，但并不是HTML
  • 它用于描述我们的UI界面，并且可以和js融合在一起使用
  • JSX在编译的时候会被转换为普通的JavaScript对象
  • 为什么React选择JSX
    • React认为，UI和逻辑应该是紧密耦合在一起的，既然使用了React来构建UI，那么逻辑也应该使用JSX来构建
    • JSX可以很好地描述UI应该呈现出它应有交互的本质形式
    • JSX在编译为JavaScript代码后进行了优化，执行更快
    • JSX是类型安全的，在编译过程中就能发现错误
    • JSX编写模板更加简单快速

# 2. JSX语法规则

  • JSX语法的本质：还是在运行的时候，被转换成了React.createElement形式来执行的
  • 书写规范：
    • jsx结构中只能有一个根元素
    • jsx结构通常会包裹一个（），将整个jsx当作一个整体，实现换行
    • jsx结构中，如果要写注释，只能这样写{/这是注释/}

# 2. React组件化开发

  • 组件化开发的好处
    • 可以让我们的代码更加模块化，也就是说一个组件的代码只负责一个功能
    • 方便复用组件
    • 方便团队开发，提高开发效率
    • 方便维护，Bug少
  • 组件化可以分为类组件与函数式组件

# 2.1 类组件(如何定义一个类组件)

- 1. 定义一个类（类名大写，组件的名称必须是大写的，小写的被认为是html元素），继承自React.Component
- 2. 在类中，必须定义一个render函数，render函数中，必须返回一个合法的JSX虚拟DOM结构

// 1. 创建组件的方式一：构造函数
// 2. 在组件中，必须返回一个合法的JSX虚拟DOM元素
// 3. 组件名称首字母必须大写
// 4. 组件的调用与原生的HTML标签一样，区别在于组件的调用是以标签的形式进行调用的
// 5. 组件的属性与原生的HTML标签一样，区别在于组件的属性值，都是字符串
// 6. 组件的属性值可以在组件内部通过this.props来获取
 // 类组件
    class App extends React.Component{
        constructor(){
            super()
            this.state={
                msg:'hello world'
            }
        }
        btnClick(){
            this.setState({
                msg:'你好，世界'
            })
        }
        render(){
            return (
                <div>
                    <h2>{this.state.msg}</h2>
                    <button onClick={this.btnClick.bind(this)}>修改文本</button>
                </div>
            )
        }
    }
    const app = ReactDOM.createRoot(document.getElementById('root'))
    app.render(<App/>)
