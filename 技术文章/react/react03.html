# 一、 setState

# 1. 为什么要使用settState

  • 因为在react中，没有做数据劫持，所以当我们修改state的值的时候，不会自动调用render方法，所以我们需要使用setState方法来修改state的值，这样就会自动调用render方法，从而更新视图。
  • 在react中，如果我们想要更新组件的状态，那么就需要使用setState方法，而不是直接修改state的值，这是因为setState方法会自动调用render方法，而直接修改state的值不会调用render方法，所以我们在修改state的值的时候，需要使用setState方法。
  • 在react中，没有做类似于vue2中的Object.defineProperty和vue3中Proxy的数据劫持，所以当我们修改state的值的时候，不会自动调用render方法，我们必须通过setState方法来修改state的值，这样就会自动调用render方法，从而更新视图。

# 2. setState的使用

  • setState方法接收两个参数，第一个参数是一个对象，第二个参数是一个回调函数，这个回调函数会在state更新完成并且视图更新之后执行。
  • setState方法是异步的，所以如果我们想要获取到更新之后的state的值，那么就需要在setState方法的第二个参数中获取。

# 3. setState异步更新的原因

  • setState方法是异步的，这是因为在react中，如果我们在一个生命周期函数中调用setState方法，那么react会把这个setState方法放到一个队列中，然后等到这个生命周期函数执行完毕之后，再去执行这个队列中的setState方法，这样就可以减少不必要的性能损耗。
  • 可以显著提升性能，如果每次调用setState都进行一次更小，那么意味着render函数会被频繁调用，界面重新渲染，这样效率很低。
  • 如果同步更新了state，但是还没有执行render函数，那么state和props就不能保持同步，这样就会导致一些问题，所以react就把setState方法放到了一个队列中，等到render函数执行的时候，再去执行这个队列中的setState方法，这样就可以保证state和props的同步。
  • 如果我们想要获取到更新之后的state的值，那么就需要在setState方法的第二个参数中获取。
  • redux设计异步的原因
  • https://github.com/facebook/react/issues/11527#issuecomment-360199710

# 4. setState在18之前与之后的区别

  • 在18之前

    • 在组件生命周期或者react合成事件中调用setState方法，那么setState方法就是异步的。
    • 在原生DOM事件或者setTimeout中调用setState方法，那么setState方法就是同步的。

  • 在18之后

    • setstate方法都是异步的，不管是在组件生命周期、react合成事件、原生DOM事件、setTimeout中调用setState方法，都是异步的。

# 二、 react性能优化SCU

  • render函数的调用
  • 在多个组件中，当多个个子组件中都单独对父组件的state里面数据有依赖，那么当父组件state的数据发生改变，那么只有对父组件有依赖数据的子组件发生改变，而不是渲染所有的子组件，调用所有子组件的render函数。
  • 解决办法：
    • 可以通过shouldComponentUpdate方法来控制调用（但是在实际开发中不常用，太过繁琐）
    • 使用PureComponent纯组件，继承PureComponent，react内部帮我们实现了。

# 1. react的更新流程

  • react在props或者state发生改变时，会调用react的render方法，会创建一颗不同的树。
  • react在基于这两个不同的树之间的差别所用到的算法进行了优化，将其优化成了O（n），如何优化的
    • 同层节点之间相互比较，不会垮节点比较
    • 不同类型的节点，产生不同的树结构
    • 开发中，可以通过key来指定哪些节点在不同用的渲染下保持稳定

# 2. 什么是SCU

  • SCU是shouldComponentUpdate的缩写，意思是是否需要更新组件，这个方法会在组件更新之前调用，如果返回true，那么就会更新组件，如果返回false，那么就不会更新组件。

  • shouldComponentUpdate有三个参数，分别是nextProps、nextState和nextContext，前两个个参数分别表示组件更新之后的props和state。

  • 当我们需要更新组件的时候，就会调用render方法，如果我们不想更新组件，那么就可以在shouldComponentUpdate方法中返回false（或者使用if判断旧值与新值），这样就不会调用render方法，也就不会更新组件。

  • 具体代码如下：

  • 父组件

import React, { Component } from "react";
import Home from "./Home";
import Footer from "./Footer";

export class App extends Component {
  constructor() {
    super();
    this.state = {
      message: "hello world",
      count: 0,
    };
  }
  AddCount() {
    // console.log("add");
    this.setState({ count: this.state.count + 1 });
  }
  changeText() {
    this.setState({
      message: "你好，世界",
    });
  }
  shouldComponentUpdate(nextProps, nextState) {
    if (this.state.message !== nextState.message) {
      return true;
    }
    if(this.state.count !== nextState.count){
      return true
    }
  }
  render() {
    console.log("App组件渲染了");
    const { message, count } = this.state;
    return (
      <div>
        <h2>App</h2>
        <h2>{message}</h2>
        <h2>{count}</h2>
        <button onClick={() => {this.changeText()}}>改变文字</button>
        <button onClick={e => this.AddCount()}>+1</button>
        <Home message={message} />
        <Footer count={count} />
      </div>
    );
  }
}

export default App;

# 三、 state---不可变的力量

# 1. 在react中切记，state是不可变的

  • 如果我们要更新state中数据的值或者状态，我们可以使用浅拷贝的方法，从原来state中拷贝一份新的对象数据，然后在再这个对象数据上增删改查，最后再把这个新的对象数据赋值给原来的数据对象。
  • 切记：千万不要在原来的数据上进行操作，会造成数据不会发生改变（因为在日常开发中：我们都是使用的PureComponent） 代码演示：
import React, { PureComponent } from 'react'

export class App extends PureComponent {
    constructor(){
        super()
        this.state ={
            students:[
                {name:'小张',age:18,grade:99},
                {name:'小李',age:20,grade:97},
                {name:'小王',age:21,grade:96},
            ]
        }
    }
    addStudent(){
        const newStudent = {name:'小陈',age:18,grade:99}
        let newStudents = [...this.state.students]
        newStudents.push(newStudent)
        this.setState({students:newStudents})
    }
  render() {
    const {students} = this.state
    return (
      <div>
        <h2>state--不可变的力量</h2>
        {
            students.map((item,index) => {
                return (
                    <div key={index}>{item.name}-{item.age}-{item.grade}</div>
                )
            })
        }
        <button onClick={()=>this.addStudent()}>添加新学生</button>
      </div>
    )
  }
}

export default App

# 2. react中如何获取dom元素

  • 在react中，如果我们想要获取dom元素，那么就可以使用ref来获取dom元素。
  • 具体有下面几种方法
    • 方式1：通过ref获取，在元素上添加ref属性，然后通过this.refs.属性名来获取
    • 方式2：通过创建createRef获取，在constructor中创建一个ref对象，然后在元素上添加ref属性，然后通过this.属性名.current来获取
    • 方式3：通过回调函数获取，在元素上添加ref属性，然后通过回调函数来获取
    • 方式4：通过forwardRef获取，在函数组件中，通过forwardRef来获取
    • 方式5：通过useRef获取，在函数组件中，通过useRef来获取
import React, { PureComponent,forwardRef } from "react";

class HelloWorld extends PureComponent {
  test(){
      console.log('组件实例内部方法')
  }
  render() {
    return <h2>hello world</h2>;
  }
}
const HelloWorld2 = forwardRef(function (props,ref) {
  return <h2 ref={ref}>hello world2</h2>;
})

export class App extends PureComponent {
    constructor(){
        super()
        this.titleRef = React.createRef()
        // 创建一个ref对象
        this.HelloWorldRef = React.createRef()
        this.threeRef = null
        // 创建函数组件ref
        this.HelloWorld2FnRef = React.createRef()
    }
  getDom() {
    // 方式1：通过ref获取
    console.log(this.refs.H2Ref);
    // 方式2：通过创建createRef获取
    console.log(this.titleRef.current);
    // 方式3：通过回调函数获取

    console.log(this.HelloWorldRef.current);
    // 执行组件内部方法
    this.HelloWorldRef.current.test()
  }
  getFnDom(){
    // 获取函数组件内部DOM
    console.log(this.HelloWorld2FnRef.current)
  }
  render() {
    return (
      <div>
        <h2 ref="H2Ref">App</h2>
        <h2 ref={this.titleRef}>方式二：创建createRef获取</h2>
        {/* <h2 ref={el => {console.log(el)}}>方式3：通过回调函数获取</h2> */}
        <h2 ref={el => (this.threeRef = el)}>方式3：通过回调函数获取</h2>
        <button
          onClick={() => {
            this.getDom();
          }}
        >获取原生DOM</button>
        <h3>ref获取类组件实例</h3>
        <HelloWorld  ref={this.HelloWorldRef}/>
        <button
          onClick={() => {
            this.getDom();
          }}
        >获取类组件DOM</button>
        <h3>获取函数组件实例</h3>
        <HelloWorld2 ref={this.HelloWorld2FnRef}/>
        <button
          onClick={() => {
            this.getFnDom();
          }}
        >获取函数组件DOM</button>
      </div>
    );
  }
}

export default App;

# 四、受控组件和非受控组件

# 1. 受控组件

  • 受控组件就是指，表单元素的值受到react的控制，表单元素的值发生改变，那么就会调用react的setState方法，然后再调用render方法，从而更新视图。

  • 受控组件的特点：

    • 受控组件的值受到react的控制
    • 受控组件的值发生改变，会调用react的setState方法，然后再调用render方法，从而更新视图
    • 受控组件的值可以通过state来获取
    • 受控组件的值可以通过setState方法来修改

  • 受控组件的优点：

    • 可以实时获取到表单元素的值
    • 可以对表单元素的值进行校验
    • 可以对表单元素的值进行修改

  • 受控组件的缺点：

    • 代码量比较多
    • 代码比较繁琐
    • 有一定的性能损耗
