# 一、TS-面向对象编程

# 1. 类的定义

  1. 类的定义：类是对一类事物的描述，是抽象的，不是具体的，对象是类的实例，是具体的。

    在 js 日常开发中，我们更加习惯于函数式编程，但是在封装某些业务的时候，类具有更强大封装性。

  2. 类的组成：---class 关键字
    1. 类的成员：属性、方法
    2. 类的属性：描述类的特征
    3. 类的方法：描述类的行为
    4. 类可以有自己的构造函数 constructor，当我们通过 new 关键字创建 一个实例时，构造函数会被调用
  3. 举例：
class Person {
  //定义属性类型
  name: string;
  age: number;
  //定义构造函数
  constructor(name: string, age: number) {
    this.name = name;
    this.age = age;
  }

  eating() {
    console.log("eating");
  }
  running() {
    console.log("running");
  }
}
//创建实例对象
const p = new Person("why", 18);
console.log(p.name);
console.log(p.age);
export {};

# 2. 类的继承与类的成员修饰符

  1. 面向对象的其中一大特性就是继承，继承不仅仅可以减少我们的代码量，也是多态的使用前提。
  2. 使用 extends 关键字来实现继承，子类中使用 super 来访问父类的构造函数和方法。
  3. 类的成员修饰符有三种
    1. public：公共的，类的内部和外部都可以访问(默认)
    1. protected：受保护的，类的内部和子类可以访问，外部不可以访问
    1. private：私有的，类的内部可以访问，子类和外部都不可以访问
  1. 举例：
class Person {
  //定义属性类型  ---- 默认就是public,在任何地方课件，公有的属性或方法
  name: string;
  age: number;
  //定义构造函数
  constructor(name: string, age: number) {}
  //private 私有成员和属性，仅在当前类中可以访问
  private eating() {
    console.log("eating");
  }
  //protected 受保护的成员和属性，仅在当前类和子类中可以访问
  protected running() {
    console.log("running");
    console.log(this.name);
    this.eating();
  }
}
const p = new Person("why", 18);
console.log(p.name);
console.log(p.age);
// p.eating();
class Student extends Person {
  constructor(name: string, age: number) {
    super(name, age);
  }
  studying() {
    console.log("studying");
  }
}
const s = new Student("why", 18);
s.studying();
// s.running();
export {};

# 3. 类的 readonly 修饰符

  1. 含义：readonly 修饰符用来修饰类的属性，表示该属性是只读的，不能被修改
  2. 举例：
class Person {
  readonly name: string;
  age: number;
  constructor(name: string, age: number) {
    this.name = name;
    this.age = age;
  }
}
const p = new Person("why", 18);
console.log(p.name);
console.log(p.age);
// p.name = '张三' readonly修饰的属性不能被修改
export {};

# 4.类的参数属性使用

  1. 在创建类的过程中，我们可以将属性直接定义在构造函数中，这样就可以省略类中定义属性的过程，同时也可以将属性修饰符直接定义在构造函数的参数上。
  2. 举例：
class Person {
  //参数属性  --- 通过在构造函数的参数前面添加一个访问限定符来声明 ---相当于语法糖
  constructor(readonly name: string, public age: number) {}
}
const p = new Person("why", 18);
console.log(p.name);
// p.name = "kobe";
export {};

# 5. 抽象类和方法的使用

  1. 抽象类：包含抽象方法(抽象方法一般没有任何的具体内容的实现)，也可以包含实例方法，抽象类是不能被实例化的，一般用来作为父类来让子类继承的。

  2. 应用场景：

    1. 抽象类中的抽象方法必须被子类实现
    1. 抽象类中的实例方法可以被子类继承
  1. 是什么？
  • 在 TypeScript 中没有具体实现的方法(没有方法体)，就是抽象方法
  • 抽象方法，必须存在于抽象类中；
  • 抽象类是使用 abstract 声明的类；
  1. 特点：
  • 抽象类是不能被实例化（也就是不能通过 new 创建）
  • 抽象方法必须被子类实现，否则该类必须是一个抽象类
  • 抽象类中的抽象方法不能有具体的实现
  1. 举例:
// 抽象类
// 定义一个抽象类，可以继承和实现
abstract class Shape {
  // 抽象类中的抽象方法不包含具体实现并且必须在派生类中实现
  //getArea方法是抽象方法，不包含具体实现，必须在派生类中实现
  //getArea方法是由声明没有实现体
  // 在方法和类前面加上abstract关键字，就可以将方法和类定义为抽象的（不能被实例化）
  // 抽象方法
  abstract getArea(): number;
}

// 定义一个派生类（继承抽象类）
//定义长方形的面积公式类和圆形的面积公式类
class Rectangle extends Shape {
  constructor(public width: number, public height: number) {
    super();
  }
  // 定义长方形的面积公式
  // 重写抽象类中的方法
  getArea() {
    console.log("长方形的面积公式为：长*宽");
    return this.width * this.height;
  }
}
class Circlec extends Shape {
  constructor(public r: number) {
    super();
  }
  getArea() {
    console.log("圆形的面积公式为：π*r*r");
    return Math.PI * this.r ** 2;
  }
}
class Triangle extends Shape {
  constructor(public a: number, public b: number, public c: number) {
    super();
  }
  getArea() {
    console.log("三角形的面积公式为：底*高/2");
    return (this.a * this.b) / 2;
  }
}
//通用的函数
// 这里也可以使用联合类型shape: Rectangle | Circle ---但是后面新增其他的图形时，就需要修改联合类型，麻烦
// 这里使用抽象类，只需要继承抽象类，然后重写抽象类中的方法即可，抽象方法不需要实现，在具体的子类中实现即可
function getShapeArea(shape: Shape) {
  return shape.getArea();
}
// 实例化  --- 传入的参数是抽象类的实例 ---多个子类继承抽象类，实例化时，传入的参数是子类的实例
// 多态 --- 一个方法可以根据传入的参数的不同，产生不同的执行效果
const rectangle = new Rectangle(10, 20);
const circle = new Circlec(5);
const triangle = new Triangle(10, 20, 30);
console.log(getShapeArea(rectangle));
console.log(getShapeArea(circle));
console.log(getShapeArea(triangle));

# 7. 类的静态成员

  1. 静态成员：在类中通过 static 修饰的属性或者方法，那么就是静态的属性或者方法，也称之为类的属性或者方法
  2. 静态成员的访问方式：通过类名.的方式来访问静态成员
  3. 静态成员的特点：
    1. 静态成员只能通过类名.的方式来访问，不能通过实例对象.的方式来访问
    2. 静态成员在类中是可以被继承的
    3. 静态成员在子类中可以被重写
  4. 举例：
class Person {
  static age: number = 18;
  static eating() {
    console.log("eating");
  }
}
const p = new Person();
console.log(p.age);
p.eating();
console.log(Person.age);
Person.eating();
export {};

# 8. TS 中的接口继承

  1. 接口的继承：接口和接口之间可以通过 extends 关键字来实现继承关系
  2. 举例：
// 接口继承
// 可以从其他接口中继承过来某一个接口的属性
// 一个接口可以继承多个接口
// 接口继承可以抽离出可重用的接口，也可以将多个接口合并成一个接口
// 接口继承使用关键字extends
// 语法格式如下：
// interface 接口名 extends 被继承的接口名 {}
// 作用：将多个接口中的共有的内容抽离成一个接口，用来给类去实现，可以实现接口的复用
// 可以减少相同代码的重复编写
// 自定义一个接口，同时又希望自定义接口拥有第三方某个类型中的所有属性
interface Animal {
  name: string;
  age: number;
  eat(): void;
}

interface Person extends Animal {
  work(): void;
}
const Person: Person = {
  name: "tom",
  age: 18,
  eat() {
    console.log("吃饭");
  },
  work() {
    console.log("工作");
  },
};
console.log(Person.name);
console.log(Person.age);
Person.eat();
Person.work();
export {};

# 二、TS-泛型

# 1. 含义：

泛型：在定义函数、接口、类的时候，不预先指定具体的类型，而是在使用的时候再指定类型的一种特性。 泛型---其本质就是类型的参数化过程，将类型当作参数一样的传递给函数，接口，类。

# 2. 举例

// 泛型的使用
// 泛型：在定义函数、接口、类的时候，不预先指定具体的类型，而是在使用的时候再指定类型的一种特性。
// 定义一个通用的函数
// function foo(arg: number) {
//   return arg;
// }
// foo(123);
// foo("123");
// 在这里面参数的类型是固定的，当传入其他类型的参数就报错，复用性不高

function bar<Type>(arg: Type): Type {
  return arg;
}
// Type 表示的是需要传递过来的具体参数参数  ---- 泛型就是将类型参数化的过程
const res1 = bar(123);
const res2 = bar<string>("56"); //完整写法<>，可以简写，根据类型推导
const res3 = bar({ name: "zs", age: 18 });
export {};

# 3. 泛型字母简称

TypeScript 中约定俗成的泛型字母简称如下： T：Type K：Key V：Value E：Element S：代表 State R：代表 Return P: 代表 Parameter O：Object 的缩写，对象 C：Callback 的缩写，回调函数

# 4. 泛型接口

在定义接口的时候我们也可以使用泛型

//  泛型接口使用
// interface Person {
//   name: string;
//   age: number;
//   habby: string;
// }
interface Person<T> {
  name: T;
  age: number;
  habby: T;
}

const person: Person<string> = {
  name: "zs",
  age: 20,
  habby: "eat",
};
const person2: Person<number> = {
  name: 1,
  age: 20,
  habby: 3,
};
console.log(person, person2);

export {};

# 6. 泛型约束

在日常开发中，有时候我们希望传入的类型具有某些共性，这个时候我们可以通过泛型约束来实现。

  1. 举例
interface Ilength {
  length: number;
}

//  1. getLength 没必要用泛型
function getLength(arg: Ilength) {
  return arg.length;
}
const length1 = getLength("aaa");
const length2 = getLength(["111", "nnn", "ccc"]);
const length3 = getLength({ length: 10 });
// const length4 = getLength(111);

//  2. 获取传入的内容，这个内容必须有length属性
function getInfo<T extends Ilength>(arg: T): T {
  return arg;
}

// const info1 = getInfo(123); //这样没有约束，number不存在length属性  可以添加extend Ilength
const info2 = getInfo("123");
const info1 = getLength(["111", "nnn", "ccc"]);
const info3 = getLength({ length: 10 });
