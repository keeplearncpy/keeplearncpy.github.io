# 一. TypeScript 简介

# 1.是什么

TypeScript 是 JavaScript 的一个超集，主要提供了类型系统和对 ES6 的支持，它由微软开发，代码开源于 GitHub 上。

# 2.为什么

# 2.1 JS 的缺点

  • JavaScript 是一种弱类型语言，这意味着它不会在编译时强制执行类型检查。这会导致在运行时出现错误，例如，如果我们将一个数字和一个字符串相加，它不会抛出错误，而是返回一个新的字符串。
  • JavaScript 是一种解释型语言，这意味着它不会在编译时执行任何操作。这会导致在运行时出现错误，例如，如果我们将一个数字和一个字符串相加，它不会抛出错误，而是返回一个新的字符串。
  • JavaScript 不支持模块化，这意味着我们不能将代码分解为独立的模块，这使得代码难以维护。
  • JavaScript 也没有加入类型检测

补充： 在开发中，错误出现的越早，越及时发现越好，这样可以减少后期的维护成本。而强类型语言在编译阶段就会报错，而不是在运行时才报错，这样就可以更早的发现错误，减少后期的维护成本。 编写发现错误 > 代码编译 > 代码运行

  • TS 是 JS 的超集，可以编译成 JS，可以在编译阶段发现错误，提高开发效率，减少维护成本。可以是使自己的代码更加健壮，更加容易维护。

# 2.2 TS 的优点

  • 加强版的 JavaScript --- 始于 JavaScript，归于 JavaScript
  • 有更好的代码提示
  • 有更完善的接口提示
  • 有更完善的类型检测
  • 不仅仅增加了类型约束，而且包括一些语法的扩展，比如枚举类型（Enum）、元组类型（Tuple）等
  • 是一个强大的工具，用于构建大型项目

# 1. 类型注解 与 类型推导

# 2. 枚举

使用 enum 关键字定义枚举类型

  1. 含义：枚举是一种特殊的数据类型，用于存储固定的数据集合
  2. 语法：enum 枚举名{枚举值 1=枚举值 1 的值，枚举值 2=枚举值 2 的值，...}
  3. 举例：
// 定义枚举类型
enum Direction {
  UP,
  DOWN,
  LEFT,
  RIGHT,
}

const d1: Direction = Direction.DOWN;
function trunDirection(direction: Direction) {
  switch (direction) {
    case Direction.LEFT:
      console.log("向左移动一步");
      break;
    case Direction.RIGHT:
      console.log("向右移动一步");
      break;
  }
}

# 3. 元组

  1. 含义：元组是一种特殊的数组，可以指定数组中每个元素的类型

  2. 语法：[类型，类型，类型]

# 4. unknown 类型

  1. 含义：用于描述类型不确定的变量
  2. 举例：
//unknown类型是ts中比较特殊的一种类型，它用于描述不确定的变量
//unknown类型默认情况下在上面进行任意操作都是非法的。any是任何操作都是合法的
//要求必须进行类型的校验(缩小),才能根据缩小之后的类型，进行对应的操作
let foo: unknown = "aaaa";
foo = 123;
if (typeof foo === "string") {
  //类型缩小
  console.log(foo.length);
}

# 5. 类型--void 类型

  1. 在 ts 中如果一个函数没有任何返回值时，那么返回值类型为 void
  2. 举例：
function fn(): void {
  console.log("hello");
}

//举个例子
//延迟执行某一个函数
type ExcFnType = (...args: any[]) => void;
function delay(fn: ExcFnType) {
  setTimeout(() => {
    fn();
  }, 1000);
}
delay((name, age) => {
  console.log(name, age);
});

# 6. 类型--never 类型

# 1. never 的应用场景
  1. 开发中很少实际去定义 never 类型
  2. 开发框架或者工具时，会定义一些工具函数，这些函数的返回值类型为 never
  3. 一般情况下，never 类型是可以赋值给任意类型的，但是没有类型是可以赋值给 never 类型的，除了 never 本身
# 2. 含义：
  1. 表示永远不会返回结果
  2. 表示永远不会结束
//封装框架工具的时候可以使用---never
function createError(message: string, code: number): never {
  throw {
    message,
    code,
  };
}

//或者
// 在开发中，其他人在扩展工具的时候，对于一些没有处理的case，可以直接报错
// 如下面的，如果给参数多添加一个boolean类型，不做case处理时，就会报错
function handleMessage(message: string | number) {
  switch (typeof message) {
    case "string":
      console.log(message.toUpperCase());
      break;
    case "number":
      console.log(message.toFixed(2));
      break;
    default:
      const check: never = message;
  }
}
handleMessage("abc");
handleMessage(123);
handleMessage(true); //never 不是string和number类型的一律报错

# 7. any 类型

  1. 在某些特定的情况下，无法确定一个变量的类型，并且可能会发生某些变化，这个时候可以使用 any 类型
  2. 当给一个标识符（变量）添加一个 any 类型是，就表示这个变量可以是任何类型，相当于 js，这个变量就失去了类型检验。
  3. 如果对于某些情况的处理过于繁琐不希望添加规定的类型注解，或者在引入一些第三方库时，缺失了类型注解，这个时候我们可以使用 any： 包括在 Vue 源码中，也会使用到 any 来进行某些类型的适配；
