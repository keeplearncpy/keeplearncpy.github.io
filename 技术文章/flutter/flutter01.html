# 2. Dart

# 2.1 为什么选Dart

  • 符合Flutter构建UI的方式，第一个使用声明式UI
  • 支持JIT(Just In Time) 和AOT(Ahead Of Time)
  • 有内存安全保证，降低了安全漏洞风险
  • 支持热重载，提高开发效率
  • 支持async和await，简化异步编程
  • 支持扩展方法，增强代码可读性
  • 支持泛型，提高代码质量
  • 支持可选类型，提高代码可读性

# 简介

  • 面向对象的编程语言(类似于Java/OC；不同于js)
  • 强类型语言
  • 线程模型
  • async 和 await异步编程
  • 泛型

# 2.2 Dart语言特性

# 1）强类型语言

  • Dart是强类型语言，变量类型一旦确定就不能修改
  • Dart支持类型推断，可以省略变量类型
  • Dart支持类型转换，但是不支持隐式类型转换

# 2）变量

  • Dart中的变量可以通过var关键字声明，也可以通过类型声明
  • Dart中的变量可以通过final和const修饰，表示常量
  • Dart中的变量可以通过dynamic关键字声明，表示动态类型

# 3）函数

  • Dart中的函数可以通过=>定义，也可以通过{}定义
  • Dart中的函数可以通过命名参数和可选参数
  • Dart中的函数可以通过匿名函数和闭包

# 4）类

  • Dart中的类可以通过class关键字定义
  • Dart中的类可以通过extends关键字继承
  • Dart中的类可以通过mixin关键字混入
  • Dart中的类可以通过with关键字实现

# 5）泛型

  • Dart中的泛型可以通过<>定义
  • Dart中的泛型可以通过extends关键字限定类型
  • Dart中的泛型可以通过mixin关键字混入类型

# 6）异步

  • Dart中的异步可以通过async和await关键字实现
  • Dart中的异步可以通过Future和Stream实现

# 7）元数据

  • Dart中的元数据可以通过@符号定义
  • Dart中的元数据可以通过@override和@deprecated使用

# 8）注解

  • Dart中的注解可以通过@符号定义
  • Dart中的注解可以通过@override和@deprecated使用

# 9）枚举

  • Dart中的枚举可以通过enum关键字定义
  • Dart中的枚举可以通过values和index获取枚举值

# 10）库

  • Dart中的库可以通过library关键字定义
  • Dart中的库可以通过import关键字导入
  • Dart中的库可以通过export关键字导出

# 11）异步

  • Dart中的异步可以通过async和await关键字实现
  • Dart中的异步可以通过Future和Stream实现

# 2.4 异步编程

  • async-await代码快是对Flutter API 的简化形式，将异步回调代码写成同步代码
  • async关键字修饰的函数返回一个Future对象，故async不会阻塞当前线程
  • await关键字会同步执行，阻塞当前线程

    var userTd = await _fetchUserId();
    var frindeId = await _fetchFrindeId(userID);
    var videoList = await _fetchVideoList(friendId);
    print('Fetch video List success');
    print('main is end');
}

    };

# 3. 两种UI和Flutter的核心

# 1）声明式UI和命令式UI

  • 命令式UI：UI界面由一个个命令操作组合而成的；
  • 声明式UI：将想要的UI描述出来即可；
//传统命令式
View view = new View();
If(color == red){
    view.setColor(red);
}else if(color == blue){
    view.setColor(blue);
}
//声明式
if(color == red){
    View.view1 = new View();
    view1.setColor(red);
    return view1;
}else if (color == blue){
    View view2 = new View();
    view2.setColor(blue);
    return view2;
}

# 4. flutter 项目目录结构和如何运行项目

  • .dart_tool 存放用于记录dart工具库所在位置及其信息的json文件和一些dary编译文件。
  • .idea 存放IDE生成的一些临时配置文件。
  • android 存放Flutter与Android原生交互的一些代码。
    • 如果需要更改 Android 应用包名、版本号、签名文件，修改方式和修改 Android 项目一样的，都是在 app 目录下的 build.gradle 里进行修改：
      android {
        compileSdkVersion 28
        defaultConfig {
            //应用包名
            applicationId "com.example.flutter_samples"
            minSdkVersion 16
            targetSdkVersion 28
            //版本号
            versionCode 1
            //版本名称
            versionName "1.0"
            testInstrumentationRunner "android.support.test.runner.AndroidJUnitRunner"
        }
}
